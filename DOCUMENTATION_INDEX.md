### 가이드 공통 구조

모든 개발 가이드는 다음 6개 섹션으로 구성:

1. **프로젝트 개요** - 기술 스택, 아키텍처 원칙
2. **코딩 컨벤션** - 명명 규칙, 코드 스타일
3. **보안 필수사항** - 보안 체크리스트
4. **생산성 가이드** - 개발 워크플로우, 도구
5. **성능 최적화** - 성능 패턴, 모범 사례
6. **Gotchas** - 흔한 실수와 해결책

## 🎯 사용 시나리오별 가이드

### 시나리오 1: 신규 프로젝트 시작

```bash
# 1. 기술 스택 가이드 선택
# Spring Boot 백엔드 + React 프론트엔드 예시
cp "templates/spring-boot/Spring-Boot-개발가이드.md" backend/CLAUDE.md
cp "templates/react/React-개발가이드.md" frontend/CLAUDE.md

# 2. 보안 가이드 추가
cp "templates/security/보안-가이드.md" docs/보안-가이드.md

# 3. 워터폴 산출물 템플릿 복사
cp -r templates/waterfall/ docs/
```

### 시나리오 2: 워터폴 SI 프로젝트

```bash
# 전체 산출물 세트 복사
cp -r templates/waterfall/ my-project/docs/
cp "templates/security/보안-가이드.md" my-project/docs/

# Claude Code 요청 예시:
# "요구사항명세서-SRS.md를 기반으로 우리 프로젝트의 SRS를 작성해줘"
# "시스템아키텍처설계서-SAD.md 템플릿으로 마이크로서비스 아키텍처를 설계해줘"
```

### 시나리오 3: 바이브코딩 학습

```bash
# 1. 바이브코딩 가이드 읽기
cat vibe-coding/README.md

# 2. 논문 분석 문서 참고
```
